Project outline

Software architecture

When thinking about a mobile robot, there are three key questions that need to be addressed.

  • Where am I?
  • Where do I want to go?
  • How do I get there?

In order to answer to these questions, the systems has to be able to:

  • Model the environment
  • Perceive the environment
  • Localise in the environment
  • Plan a trajectory and execute the movement

The simple robot demonstrated in the video is addressing the previous topics by

  • Perceiving the environment using a laser range sensor and a camera
  • Model the environment as straight line features on a map.
  • Localise using exteroceptive(laser) & proprioceptive(wheel encoders) measurements and Extended Kalman Filtering.
  • Plan a path using wavefront-planner.
  • Execute the path using a piecewise stable motion controller.

Environment perception & modelling

The world is described by hand-crafter features, that we as humans perceive as lines. The robot is extracting line features from the raw laser-point data by running a RANSAC algorithm.

Localisation

The robot is constantly using its kinematic model (differential drive) along with wheel encoder measurements to predict its traveled trajectory based on odometry, it is widely known that odometry world estimates are drifting due to a lot of effects which are either on purpose not being considered on the motion model, or they are not known. Extended Kalman filtering can reduce the estimate error of the robot's pose in the environment with respect to a a-priory known map of feature points.

Motion controller

The path planner is creating wheel velocity commands, based on a relative target pose. The controller is designed in polar coordinates as this provides a suitable framework to design an asympotically stable feedback controller.
