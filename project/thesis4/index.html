Thesis report Week#7

Mar 15, 2020

Tracking a single target in clutter

Problem formulation:

Perform state estimation of a single target using observations of uncertain origin.

  • Target's tracked state at sample time $k$ is, $\mathbf{x}_k=(x,y,V,h,\omega)^T$ with respect to the self-vehicle sea-keeping frame.

  • Assume its track is initialized tracks i.e the problem is that of track maintenance .

  • Observations $z_k = h(\mathbf{x}_k) + w_k$ from various sensors, defined my a corresponding measurement model and the target's state.

  • $w$ is gaussian white and its covariance matrix depends on the sensor properties.

  • Detection probability $P_D$ less than unity, i.e. there exists the possibility of the sensor not returning an observation that belongs to the target, due to various reasons.

  • Non zero $P_{FD}$, ie the sensor can provide false positive observations that do not originate from the tracked target, usually due to clutter.

  • Possibility for multiple validated observations for a single target.

Clutter model

The target tracker estimates the target state such as position and orientation from the noisy measurement observations originating from various object's sources such as real targets, clouds, terrain reflections and thermal noise. The unwanted measurements are referred as clutter measurements.[source]

Most of the sensors are finite in their perception resolution, that is they have $N$ resolution cells. A detection is declared in a cell if the output of the signal for the specific cell exceeds a threshold. If the sensor points to a region which is not occupied by a target, then false detections can still occur due to sensor noise or background noise. Usually filtering is performed with observations instead of detections, that is with the output of the raw-data preprocessor.

In radar tracking for instance, the preprocessor might run clustering algorithms on the raw detections and provide the mean of each cluster as a different observation. The preprocessing of the raw data should remove what was defined as clutter, nevertheless preprocessors still produce false positive observations. Observations(post-processed information from raw data) can have a similar clutter model.

Assumptions

  • The probability of a false positive observation inside the sensors grid cell is $P_{FA} = p$ in each cell
  • The events of observations in each cell are independent of each other.

Then the probability mass function (pmf) of the number of false alarms in these $N$ cells, follows a binomial(Bernouli) distribution

\[ P\{n_{FA}=m\} = \mu_{FA}(m) = \begin{pmatrix}N \\m \end{pmatrix} p^m (1-p)^{N-m} \]

If $V$ is the volume of the $N$ cells under consideration, then $\lambda$ is the spatial density of the false alarms

\[ \lambda = \frac{E{n_{FA}}}{V} = \frac{Np}{V} \]

If $Np >1$ then the Poisson distribution can be approximated by

\[ \mu_{FA}(m) \approx e^{-Np}\frac{(Np)^m}{m!} \]

thus the number of false alarms(clutter observations) in a volume $V$ given a spatial density of clutter observations $\lambda$ is

\[ \mu_{FA}(m) \approx e^{-\lambda V}\frac{(\lambda V)^m}{m!} \]

The justification of using the above clutter model, is that the pdf of a false observation is uniform

\[ p(z|{z = \text{false measurement}}) = \frac{1}{V} \]

where $V$ corresponds to the subspace volume in which the observation is originated from.

Approaches to filtering

  • Non-Bayesian techniques
    • Nearest Neighbour association
    • Strongest Neighbour association (assuming intensity information)
    • Track-Split approach
  • Bayesian techniques
    • Probabilistic Data Association filter
    • Optimal Bayesian approach, multiple hypothesis (MH)

Probabilistic Data Association Filter (PDAF) overview

This filter is based on the KF/EKF algorithm, it keeps the update equations the same but is probabilistically fusing observations within the target's validating region into a single fused innovation update step.

Modeling assumptions of the PDAF

  • Single target of interested modeled by white-noise driven state motion modele as in KF/EFK
  • The target's track is already initialized
  • At each sensor sample time, a validation region(gate) to filter the highly unprobable observations is created.
  • The target exhibits a detection probability at eash sensor scan $P_D$
  • The remaining observations(not originating from the target) are assumed due to false detection or clutter, and are assumed to be of uniform spatial distribution.
  • All past information about the target is summarised by a single state estimate $\hat{\mathbf{x}(k-1|k-1)}$ and its associated covariance $P(k-1|k-1)$
  • The observation pdf originating from the target is Gaussian with mean $\hat{z}(k|k-1)$ and innovation covariance $S(k)$.

PDAF algorithm

Two versions exist

  • Non parametric
    • A diffuse prior is used as knowledge for the number of false measurements
    • This requires no knowledge about the spatial density of false measurements
  • Parametric
    • A Poisson pmf is used for the number of false measurements.
    • Requires the spatial density parameter $\lambda$ (expected number per unit volume) of the false measurements.

Prediction

Predict the state and the observation:

\[ \begin{align} \hat{\mathbf{x}}(x+1|k) &= F(k)\hat{\mathbf{x}}(k|k) \\ \hat{z}(k+1|k) &= H(k+1)\hat{\mathbf{x}}(x+1|k) \end{align} \]

Calculate the predicted state covariance

\[P(k+1|k) = F(k)P(k|k)F(k)^T + Q(k)\]

The innovation covariance is

\[S(k+1) = H(k+1)P(k+1|k)H(k+1)^T + R(k+1)\]

The above steps are the same as in the standard KF/EKF.

Observations validation

A validation region(gate) is defined by the ellipsoid:

\[ \mathcal{V}(k+1, \gamma) = \{z: [z-\hat{z}(k+1|k)]^T S(k+1)^{-1} [z-\hat{z}(k+1|k)] \leq \gamma \} \]

where $\gamma$ is the gate threshold and can be calculated by the chosen gate probability $P_G$ and the chi-squared distribution lookup matrix.

This leads to a validated observations set

\[ \{z_i(k+1), i = 1, ..., m(k+1)\} \]

The innovation $\nu_i$ corresponding to the i-th validated observation is

\[\begin{align} \nu_i(k+1) = z_i(k+1) - \hat{z}(k+1|k) && i= 1, ...,m(k+1) \end{align} \]

The validation region volume $V$ is

\[V(k+1) = c_{n_{z}}|\gamma S(k+1)|^\frac{1}{2}\]

where $c_{n_z}$ is the volume of the $n_z$ dimensional unit hypersphere

\[ c_i = \left\{\begin{array}{ll} 2 & i = 1 \\ \pi & i=2 \\ \frac{4\pi}{3} & i = 3 \end{array} \right. \]

Probabilistic Data Association ~ non parametric version

The probability that the i-th measurement is the correct one can be calculated

\[\beta_i(k+1)= \left\{\begin{array}{ll} \frac{e_i}{b+\sum_{j=1}^{m(k+1)}e_j} & i = 1,...,m(k+1) \\ \frac{b}{b+\sum_{j=1}^{m(k+1)}e_j} & i=0 \end{array} \right. \]

where $\beta_0$ corresponds to the probability that none of the measurements is correct

The likelihood $e_i$ is

\[ e_i \overset{\Delta}{=} e^{-\frac{1}{2} \nu_i(k+1)^T S(k+1)^-1 \nu_i(k+1)} \]

\[ b \overset{\Delta}{=} \left(\frac{2\pi}{\gamma}\right)^{\frac{n_z}{2}}m(k+1)c_{n_z}^-1\frac{1-P_D P_G}{P_D} \]

Update

The state update can be carried by first calculating the weighted combined innovation

\[ \nu(k+1) \overset{\Delta}{=} \sum_{i=1}^{m(k+1)}\beta_i(k+1)\nu_i(k+1) \]

Then the state update follows

\[ \hat{x}(k+1|k+1) = \hat{x}(k+1|k) + K(k+1)\nu(k+1) \]

where the Kalman gain $K(k+1)$ follows the standard EKF/KF calculation.

The updated covariance is

\[ P(k+1|k+1) = P(k+1|k) - [ (1-\beta_0(k+1)) ] K(k+1)S(k+1)W(k+1)^T + \tilde{P}(k+1) \]

where $\tilde{P}(k+1)$ is responsible for spreading the updated estimate variance(increase uncertainty)

\[ \tilde{P}(k+1) \overset{\Delta}{=} K(k+1)\left[ \sum_{i=1}^{m(k+1)}\beta_i(k+1)\nu_i(k+1)\nu_i(k+1)^T - \nu_i(k+1)\nu_i(k+1)^T \right]K(k+1)^T \]

Remarks on PDAF

  • $\tilde{P}(k+1)$ is positive definite and increases the covariance of the updated state, and is capturing the effect of origin uncertainty.

  • The estimation accuracy is data dependend.

  • In the parametric PDA , $\beta$ is calculated by replacing $m(k+1)$ with $\lambda V(k+1)$ , where $\lambda$ is the spatial density of the false measurements of the clutter model.

    The distinctive difference between the parametric and the non-parametric variants, is that the parametric asssumes an apriori known diffusion density $\lambda$ , where as the non-parametric one is calculating it based on the current number of validated observations $m(k+1)$ and the validation volume $V$.

  • While the structure is the same as in a linear filter, PDAF is a non-linear filter due to the weighted innovation calculation.
